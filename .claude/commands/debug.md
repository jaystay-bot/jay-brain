Systematic debug protocol. Use when something is broken.

1. Read lessons.md first — has this happened before?
2. Check logs — Vercel runtime logs, browser console, terminal
3. Identify the exact error message — copy it word for word
4. Identify which file and line number
5. Identify root cause — not the symptom
6. Write the fix plan before touching any code
7. Make one change at a time
8. Test after every single change
9. Confirm it works end to end
10. Add the lesson to lessons.md
Never patch symptoms. Never change multiple things at once.
